# 테스팅 가이드

Phase 6 완료: E2E 테스트 및 에러 처리 개선

## 🎯 개요

이 가이드는 Habit Quest 자녀용 앱의 전체 기능을 테스트하는 방법을 설명합니다.

## 🛠️ 테스트 환경 설정

### 1. 백엔드 서버 시작

```bash
# PostgreSQL & Redis 시작
cd /Users/dragon/IdeaProjects/game
docker-compose up -d postgres redis

# API 서버 시작
cd apps/api
source venv/bin/activate  # 가상환경 활성화
uvicorn app.main:app --reload
```

✅ API 서버: http://localhost:8000
✅ Health Check: http://localhost:8000/health

### 2. 프론트엔드 개발 서버 시작

```bash
cd apps/web-child
npm run dev
```

✅ 개발 서버: http://localhost:3001

## 🧪 빠른 테스트 (5분)

### 회원가입 → 로그인 → 대시보드

1. **회원가입**
   - http://localhost:3001/register 접속
   - 이메일: `test-$(date +%s)@test.com`
   - 비밀번호: `password123`
   - 닉네임: `테스터`
   - 회원가입 클릭 → 페어링 페이지로 이동

2. **페어링 스킵**
   - "나중에 하기" 클릭 → 대시보드로 이동

3. **대시보드 확인**
   - 캐릭터 정보 표시 확인
   - HP/EXP 바 애니메이션 확인
   - 골드, 보석, 스탯 확인

4. **퀘스트 확인**
   - 하단 "퀘스트" 탭 클릭
   - 습관 목록 표시 확인
   - 새 습관 추가 테스트

## 📝 전체 플로우 테스트 (20분)

### Phase 3: 습관 관리

#### Test 3.1: 습관 체크
```
1. /quests 페이지 접속
2. 미완료 습관 체크박스 클릭
3. 로딩 인디케이터 표시
4. 보상 토스트 (골드, EXP) 확인
5. 캐릭터 정보 업데이트 확인
```

#### Test 3.2: 새 습관 추가
```
1. "새 습관 추가" 버튼 클릭
2. 이름: "매일 운동하기"
3. 카테고리: 운동
4. 난이도: 보통
5. "추가" 클릭
6. 목록에 추가 확인
```

#### Test 3.3: 레벨업
```
전제조건: 레벨업에 필요한 EXP 근처
1. 습관 여러 개 체크
2. 레벨업 모달 표시 확인
3. 새 레벨, 스탯 증가 확인
4. 모달 닫기 → 정보 업데이트
```

### Phase 4: 보스전

```
전제조건: 캐릭터 레벨 5, 10, 15...

2. 전투 탭 클릭

5. 보스 턴 → 데미지 받음
6. 마나 소모 확인
```

#### Test 4.3: 승리
```
1. 보스 HP 0 도달
2. 승리 화면 표시
3. 보상 (골드, EXP, 보석) 확인
4. "보상 받기" 클릭
5. 캐릭터 정보 업데이트
```

## ⚠️ 에러 처리 테스트

### Network Errors

#### Test E1: API 서버 다운
```
1. API 서버 중지
2. 습관 체크 시도
3. "서버에 연결할 수 없습니다" 메시지 확인
```

#### Test E2: 느린 네트워크
```
1. Chrome DevTools > Network > Slow 3G
2. 페이지 로드 시 로딩 스피너 확인
3. 30초 타임아웃 후 에러 메시지
```

#### Test E3: 오프라인 모드
```
1. Chrome DevTools > Network > Offline
2. 상단에 오프라인 배너 표시 확인
3. 온라인 복구 시 "다시 연결" 메시지
```

### Authentication Errors

#### Test E4: 토큰 만료
```
1. 로그인 후 localStorage의 token 삭제
2. 보호된 페이지 접속 시도
3. 자동 로그아웃 및 /login 리다이렉트
```

### Validation Errors

#### Test E5: 입력 검증
```
회원가입/페어링/습관추가에서:
1. 빈 필드
2. 잘못된 형식
3. 너무 긴/짧은 입력
→ 모두 적절한 에러 메시지 표시
```
